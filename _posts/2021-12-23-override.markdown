#### 方法重写/覆盖（override）：



方法重写需要满足以下条件：

1.子类的方法的参数，方法名称，要和父类方法4的参数、方法名称完全一样。

2.子类方法的返回类型和父类方法的返回类型一样，或者是父类返回类型的子类。举例：父类返回类型为Object，子类返回类型为String，String是Object的子类

3.子类的方法不能缩小父类方法的访问权限，`public` >  `protected`  >  `默认` >  `private`    **可以放大或者相等**



| 名称 | 发生范围 | 方法名   | 形参列表                 | 返回类型           | 修饰符                     |
| ---- | -------- | -------- | ------------------------ | ------------------ | -------------------------- |
| 重载 | 本类     | 必须相同 | 类型个数顺序至少一个不同 | 无要求             | 无要求                     |
| 重写 | 父子类   | 必须相同 | 相同                     | 子父类相同或者包含 | 子类不能缩小父类的访问范围 |





代码复用性不高，不利于代码维护

#### 多态：方法或者对象具有多种形态，是面向对象的第三大特征，多态是建立在封装和继承的基础上的。

##### 对象的多态※：

1. 一个对象的编译类型和运行类型**可以不一致**。
2. 编译类型在定义对象时就确定了，不能改变。
3. 运行类型时可以变化的。
4. 编译类型看定义时=号的左边，运行时看=号的右边。
5. `Animal animal = new Dog()`   animal编译类型`（javac）`时Animal，运行类型`（Java）`是Dog

多态的细节讨论：

首先，两个类具有继承关系，其次是，向上转型：

#### 向上转型：

**父类的引用指向子类的对象**     `父类类型引用名 = new  子类类型()`

**可以调用父类中的所有成员（须遵守访问权限，私有的无法访问），但是不能调用子类的特有成员，**因为在编译阶段可以调用哪些成员是由编译类型决定的，编译类型看等号左边，等号左边为父类，所以可以调用父类，但不能调用子类特有的。

**最终的运行效果还是看子类的实现**。如下：  `Animal animal = new Cat()`

`animal.eat()`    运行时，交给Cat()来运行，运行调用方法时看右边，右边时先看`Cat()`类型，依次往其父类上寻找。

#### 向下转型： （我需要获取子类的特有方法，采用这个方法）

`Cat cat = (Cat) animal`   编译类型就是Cat   运行类型就是右边Cat 

- `子类类型 引用名  =  (子类类型) 父类引用`     

- 只能强转父类的引用，不能强转父类的对象

- **要求父类的引用必须指向的时当前目标类型的对象**   

  要求animal本来就是准备指向Cat `Animal animal = new Cat()`

- 向下转型后。可以调用子类类型中所有的成员。



##### **※没有属性重写：属性没有重写的说法，属性的值就看编译类型（等号左边），编译类型是什么就是什么**

- `instanceof`比较操作符，用于判断对象的**运行类型**是否为XX类型或XX类型的子类型。

#### ※※Java动态绑定机制：

Java的动态绑定机制：**1.当调用对象方法时，该方法会和该对象的内存地址/运行类型绑定**

2.当**调用对象属性**时，没有动态绑定机制，**哪里声明哪里使用**。

多态应用：

**多态数组：数组的定义类型为父类类型，实际元素类型为子类类型。**

数组强转为子类类型：  （`instanceof`）

```java
if(pesons[i] instanceof Student){
    ((Student)person[i]).study();
}
```

`study()`是学生子类的特有方法

**多态参数：方法定义类型为父类类型，实参类型允许为子类类型**

```java
    public void testWork(Employee e){
        if(e instanceof worker){
            ((worker)e).work();
        }else if(e instanceof manager){
            ((manager)e).mangage();
        }
    }
```
