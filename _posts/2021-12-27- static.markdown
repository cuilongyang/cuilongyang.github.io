#### 类变量 / 静态变量：

静态变量/静态属性，任何一个对象去访问它时，得到的都是相同的值，同样的任何一个该类的对象去修改它时，修改的也是同一个变量。举例：

```java
class Child {
    private String name;
    //定义一个类变量，(静态变量)static静态
    //该变量的最大特点就是会被Child类的所有对象实例共享
    public static int count = 0;

    public Child(String name) {
        this.name = name;
    }

    public void join() {
        System.out.println(name + "加入了游戏");
    }
}

```

- 静态变量被同一个类的所有对象共享的，无论其是在**堆里**还是方法区的静态域里，都不会影响使用。

- static变量在类加载的时候已经生成了。保存在class实例的尾部，class实例保存在堆中。

#### 定义语法：

  ```java
  访问修饰符 static 数据类型 变量名;（推荐这个方法）
  static 访问修饰符 数据类型 变量名;
  ```

#####   访问类变量：举例如下：

```java
public class VisitStatic {
    public static void main(String[] args) {
        //类名.类变量名
        //类变量随着类的加载就创建，即使没有创建实例也可以访问
        System.out.println(A.name);
    }
}
class A{
    public static String name ="belle";
}
```

#### 类变量使用细节：

1. 什么时候需要类变量？需要某个类的所有对象共享一个变量时，就可以考虑使用类变量，举例有多个学生，统计学生一共交多少钱，交钱这个费用可以使用类变量。
2. 类变量和实例变量区别：类变量时该类的所有对象共享的，而实例变量是每个对象独享的。
3. 加上static成为类变量或静态变量，否则称为实例变量/普通变量/非静态变量。
4. 类变量可以通过`类名.类变量`名或者 `对象名.类变量`名来访问，推荐使用前者，但需满足访问修饰符的访问权限和范围。
5. 实例变量**不能**通过`类名.类变量` 来访问。
6. `static变量`在类加载的时候已经初始化了。
7. 类变量生命周期随着类的加载开始，随着类的销毁而结束。

#### 类方法：

当方法使用了static修饰后，该方法就是静态方法，静态方法可以调用静态变量/属性。

**如果我们希望不创建实例也可以调用某个方法（即当作工具来使用），此时将此作为静态方法使用。**

```java
 System.out.println("9开平方的结果是：" + Math.sqrt(9));
```

此时的 `Math.sqrt()`  就是采用静态方法，**可以作为工具使用。**（开发自己的工具类，作为方法使用）

#### 类方法使用细节：

1. 类方法和普通方法都是随着类的加载而加载，将结构信息储存在方法区：类方法中无this的参数，普通方法中包含着this参数。

2. 类方法可以通过类名调用，也可以通过对象名调用。

3. 普通方法和对象有关，需要通过对象名调用，不能通过类名调用。

4. 类方法中不允许使用和对象有关的关键词，比如this super。普通方法可以使用。

5. 类方法（静态方法）中只能访问静态变量和静态方法。

6. 普通成员方法既可以访问普通变量/方法，也可以访问静态变量/方法。

总结：**静态方法只能访问静态成员**，普通方法可以访问静态、普通的成员都可以
